Visual Editor

Three-panel workspace,
zero boilerplate

Lucy Studio gives you a Figma-like editing experience that outputs real Flutter code. Left panel for widgets, center canvas for live preview, right panel for properties and data binding.

  • Left panel — widget library organized by category
  • Center canvas — device frame with real-time rendering
  • Right panel — Property inspector & Data binding tab
  • Multi-device preview: phone, tablet, responsive
Widget Library

Every Flutter widget,
ready to drop

20+ production-ready widgets organized into Layouts, Contents, and Dividers. From simple Text and Image to complex TDTable, DragList, and NestedScroll — no code needed for most UI patterns.

  • Layout primitives: Row, Column, Stack, Grid, ListView, PageView
  • Advanced layouts: TDTable, TDGrid, DragList, NestedScroll, Masonry
  • Content widgets: Text, TextField, Image, SVG, Button, Checkbox
  • Responsive helpers: Safe Area, VSplit, HSplit, Wrap
Animation & States

State-machine driven
animation editor

Define multiple visual states per widget — Default, Hover, Pressed, Error, Loading — and set transition animations between them without writing a single setState.

  • Named state groups: define unlimited states per widget
  • Timeline-based transitions — duration, curve, delay per property
  • Condition-based state switching linked to data or user gestures
  • Full animation mode — boards, components, and interactions
Figma Import

From Figma to Flutter
in minutes

Lucy Studio bridges the design-to-development gap. Install the Figma Color Importer plugin, copy JSON from Figma, and paste directly into Lucy Studio. Colors, tokens, pages, icons — all imported instantly.

  • Figma Color Importer plugin — 3-step setup
  • Light / Dark mode color variables imported simultaneously
  • Pages, icons, and component references synchronized
  • Token names auto-mapped to Lucy variable names
Data Binding

Connect UI to real data
without code

Bind any widget property directly to an API endpoint, data provider, or app variable. Configure data sources, set call parameters, and map response fields to widget properties — entirely visually.

  • Contents Datasource — API endpoint per widget with call configuration
  • Provider Instance — shared data providers reused across pages
  • Data Set — custom static or mock datasets for prototyping
  • Drag-and-drop field mapping from response JSON to widget properties
Variables

Design tokens built in
to every project

Manage app-wide variables and color tokens in one centralized panel. Changes propagate instantly to every widget that references them — the same token system as your Figma file.

  • Color Variables with full Light / Dark mode support
  • App Variables — String, Boolean, Number, List scopes
  • Page-scoped or global across the entire project
  • Import from Figma or define from scratch
Pages & Navigation

Multi-page apps with
zero routing code

Organize your app's screens in a hierarchical folder tree. Set navigation actions on any widget — push, replace, pop — and Lucy handles Flutter routing under the hood.

  • Folder-based page tree for complex multi-screen flows
  • Named route management with parameter passing
  • Widget tree view per page for quick inspection
  • Recent pages panel for fast context switching
OTA Deploy

Push updates instantly
without app store review

Deploy UI changes directly from Lucy Studio to production devices via Lucy's OTA infrastructure. No recompile. No app store wait. Configure your deploy target, hit upload, and users see the update in seconds.

  • One-click deploy to Lucy Deploy server from inside Studio
  • Target specific environments: dev, staging, production
  • Deploy Uploader Settings — server IP, credentials, sync options
  • Rollback to any previous deployment at any time
Console & Debug

Full visibility inside
the editor

Lucy Studio's built-in console surfaces Studio logs and App logs side by side. Trace data binding calls, inspect widget state changes, and catch runtime errors without ever leaving your workspace.

  • Studio Log — editor-level events, build steps, deploy status
  • App Log — runtime Flutter output from connected device or emulator
  • Click any log entry to jump to the originating widget
  • Filter by level: Info, Warning, Error, Success
